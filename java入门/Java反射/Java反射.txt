java 的反射机制
在反射机制运行的过程中，对于任意的一个对象可以使用它的任意方法和属性，对于一个类能够知道它的所有方法和属性。

Class.forName(),类的实例名.getClass,类名.class
三个方法得到的都是java.lang.Class对象。不同的地方在于
1)Class cl=A.class; JVM将使用类A的类装载器,将类A装入内存(前提是:类A还没有装入内存),不对类A做类的初始化工作.返回类A的Class的对象。

2)Class cl=对象引用o.getClass();返回引用o运行时真正所指的对象(因为:儿子对象的引用可能会赋给父对象的引用变量中)所属的类的Class的对象 。

3)Class.forName("类名")，装入类A,并做类的初始化。

看到初始化我稍微有些疑惑，在我的理解中初始化就是对类进行实例化。然而并不是这样。
在java中使用任意的一个类都需要经过装载、连接、初始化这三个过程。


初始化是为类的静态变量赋予初始值，并会执行类中的静态代码块。
newInstance，获得一个类的实例
newInstance: 弱类型。低效率。只能调用无参构造。 
new: 强类型。相对高效。能调用任何public构造。
